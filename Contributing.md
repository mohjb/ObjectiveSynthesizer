# Contributing to Objective Synthesizer
Thank you for your interest in contributing to the Objective Synthesizer project! Your contributions are what make this a powerful and versatile tool for the community. This document outlines the guidelines for contributing to the project.
Ways to Contribute
There are many ways you can help, even if you don't write code:
 * Report Bugs: If you find a bug, please report it so we can fix it.
 * Suggest Enhancements: Have an idea for a new feature or improvement? Let us know!
 * Improve Documentation: Found a typo or an unclear explanation in the README.md or other files? Help us make it better.
 * Write Code: Implement new features, fix bugs, or improve the project's performance.
Getting Started
The project is structured as a single HTML file for ease of use. To get started:
 * Clone the repository: git clone https://github.com/mohjb/ObjectiveSynthesizer.git
 * Open the index.html file in your preferred code editor.
 * You can view and test the project by opening index.html directly in a modern web browser.
Reporting Bugs
When reporting a bug, please open a new issue on the GitHub repository and include as much detail as possible:
 * Clear, descriptive title: Summarize the issue concisely.
 * Steps to reproduce: Provide a numbered list of steps that reliably trigger the bug.
 * Expected behavior: Describe what you expected to happen.
 * Actual behavior: Describe what actually happened.
 * Screenshots/Gifs: Visual aids are extremely helpful.
Suggesting Enhancements
If you have a feature request or an idea for a new component, please open a new issue with a clear title and description. Explain the problem your idea would solve and how it would improve the user experience.
Submitting a Pull Request (PR)
If you want to contribute code, please follow these steps:
 * Fork the repository and create a new branch from main: git checkout -b feature/your-feature-name or git checkout -b bugfix/your-bug-name.
 * Make your changes. Be sure to update the relevant code within the index.html file.
 * Test your changes to ensure they work as expected and don't introduce new bugs.
 * Write a clear commit message. We follow the Conventional Commits specification to help with changelog generation.
   * feat: add new component
   * fix: resolve bug in button styling
   * docs: update readme with new feature
 * Push your branch to your forked repository.
 * Open a Pull Request against the main branch of the official repository. Describe your changes clearly and link to any related issues.
Code of Conduct
We are committed to fostering a welcoming and inclusive community. Please be respectful and considerate of other contributors. All participants are expected to follow the Code of Conduct.
Thank you for your help in building this project!
